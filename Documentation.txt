Barre de recherche centralisée



But:

Cette fonctionnalité permet de rechercher une œuvre en particulier dans la base de données à l’aide d’un type précis de recherche et selon l’information que l’utilisateur possède en main. 

La barre de recherche est située toutes les pages, elle est donc facile d’accès.

Utilisation: 

L’utilisateur devra donc en premier lieu dérouler la barre de recherche en appuyant sur une petite flèche vers le bas, et sélectionner le type de recherche qu’il veut faire à l’aide d’un input select, soit par artiste, arrondissement, ou catégorie. 

Suite au premier choix, un deuxième select contenant les arrondissements ou catégories disponibles, ou bien un input de type texte avec auto-complete apparaîtra pour finaliser la recherche. 

Finalement, l’utilisateur devra appuyer sur le submit pour accéder à la page de résultats.

Design:

Le design de la barre de recherche se veut simple, en rapport avec le reste du site. Les couleurs seront les mêmes que celles choisies pour le site, afin de se marier et de ne pas surcharger la page.

Programmation:

La barre de recherche sera programmée en PHP/Javascript, et sera déroulée à l’aide de JQuery. 

Le choix du type sera effectué à l’aide d’un input type sélect, et qui affichera la seconde partie de la recherche suite à un submit via un onchange.

Un deuxième sélect, rempli cette fois avec PHP pour recueillir les informations de la base de données, offrira des arrondissements avec la location des œuvres ou une catégorie spécifique reliée à l’œuvre recherchée.

Dans le cas d’un nom d’artiste, un auto-complete fait à l’aide de JQuery offrira des résultats existants à l’utilisateur selon les lettres qu’il entrera dans le Input texte.

Une requête filtrera les résultats selon l’information fournie par l’utilisateur.

Gestion des œuvres (ajout)


But:

Cette fonctionnalité permet à un utilisateur et un administrateur du site de soumettre une nouvelle œuvre sur le site de Montreart, afin de rendre la base de données du site la plus complète et à jour possible.

Utilisation:

L’utilisateur/administrateur doit d’abord passer par un formulaire d’ajout lui demandant d’entrer les informations de base concernant l’œuvre, étant le titre, l’artiste, l’adresse, une description, la catégorie et l’arrondissement dans lequel l’œuvre se trouve.

Design:

Le design de la page de gestion n’est pas trop chargé pour ne pas entrer en conflit avec le formulaire, qui se veut le plus clair possible. Le thème et les couleurs se marient avec le reste du site.

Programmation:

À l’aide de requêtes, les informations seront soumises dans la base de données après l’approbation de l’administrateur du site. Une validation des champs supprime les espaces vides et requiert une chaîne de caractères dans ceux-ci.

Accepter ou refuser une soumission utilisateur

L’onglet « Approuver les soumissions » dans la page de gestion permet de visualiser toutes les soumissions nécessitant d’être approuvées par un administrateur avant d’être rendues visible sur le site. Il s’agit des soumissions d’œuvres, de photos et de commentaires. Lorsque vous accédez à cet onglet, cliquez sur une des sections pour visualiser les liens vers les soumissions de la catégorie. La mention « En attente : » suivie d’un chiffre à la droite des onglets permet de visualiser le nombre de soumissions en attente d’une approbation sans avoir à dérouler l’onglet. 
Lorsque vous cliquez sur un lien vers une soumission en attente, un panneau affichant les informations soumises par l’utilisateur s’affiche. Deux boutons vous permettent soit d’accepter ou refuser la soumission. Si vous l’acceptez, elle sera désormais instantanément visible pour tous les visiteurs du site. Si vous cliquez sur refuser, la soumission sera supprimée de la base de donnée. Si la requête est réussie, le panneau se ferme et le nombre de soumissions restantes se met à jour. Dans le cas contraire, le panneau restera ouvert et un message d’erreur indiquant la nature du problème s’y affichera.
Dans chacun des affichages de soumission, il est possible de cliquer sur le titre de l’œuvre correspondant à la soumission dans le haut du panneau d’affichage. Ce lien vous redirigera vers une nouvelle page affichant toutes les informations de l’œuvre telle qu’affichée sur le site actuellement.

Vue Trajet

But:

L'utilisateur sera capable de visualiser sa localisation sur la rate et les œuvres d'art public les plus proches. A partir de sa position ou de un point de départ tapé dans le premier champ du formulaire, il sera capable de se faire un itinéraire pour une promenade dans son proximité.

Utilisation: 

L'utilisateur laissera les coordonnées déjà remplis par default, ou il tapera une adresse comme point de départ. Ensuite il choisira une des œuvres d'une select comme destin. Finalement il choisira d’un select multiple les œuvres qu'il désire comme arrêts intermédiaires. En envoyant le formulaire, une route se tracera sur la carte, et un texte contenant la description de sa route apparaitra en dessous de la carte.

Programmation:

Utilisation de l'API Google Maps Directions. Il est de noter que ceci ne prend pas la géolocalisation en temps réel, c’est-à-dire, il n'est pas capable de changer le point de départ, une fois envoyé la requête. On a employé la formule Haversine pour calculer les œuvres les plus proches a l'utilisateur dans un rayon déterminé. Ce rayon est présentement en fonction du zoom de la carte Google.

Se connecter

Pour se connecter au site, il faut appuyer sur l’onglet « se connecter » dans la barre de navigation. Le nom d’usager et le mot de passe doivent être entrés exactement comme lors de l’inscription, incluant les majuscules, sans quoi l’accès sera refusé. Un utilisateur régulier connecté au site pourra visualiser et modifier sa page de profil, en plus d’accumuler des points en fonctions des œuvres visitées. Un utilisateur de type administrateur aura en plus accès à la page gestion pour interagir avec la base de données. Une fois connecté, les liens pour accéder à ces fonctionnalités s’ajouterons à la barre de navigation.
